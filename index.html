JavaScript Simon Game

JavaScript Simon Game

Web Based Game App

This project's objective was to recreate the classic game, Simon, into a digital game application.

  • Built with HTML, CSS, jQuery and JavaScript
  • The game's logic will generate a random sequence, unique to each game play.
  • The player has to repeat the sequence shown by the game's flashing colors and sounds.
  • As the player progresses through the game, the speed of the game's sequences will increase.
  • Optional "strict mode" for players who wish to challenge themselves. If a player fails to repeat the game's sequence every time, the game will restart from level 1.
  • To win the game, the player must complete 20 levels.

Twitch.tv API Viewer

Twitch.tv Channel Viewer

Utilizing Twitch.tv's API

This project's objective was to use Twitch.tv's API and create a single page, channel viewer. The channels were pre-selected based on what channels were followed by one particular user.

  • The viewer shows if a channel is online or offline.
  • By default, all online channels are shown first, offline channels are secondary and unavailable channels are shown last.
  • A simple set of buttons on top allows fast and easy switching of views to see online and/or offline channels.
  • If a channel is online, a description of the content being streamed is provided.
  • Each channel presented is clickable by users and links directly to the actual channel on Twitch.tv.
  • The interace was based on TV programming guides that are popular among most people with cable or satellite TV.
  • The layout, even though simple, is responsive to various devices and screen sizes.
Wikipedia Search

Wikipedia Search

via Wikipedia's API

This project was to create a simplified search using Wikipedia's API. The interface was designed to emulate a typical search engine, with search results appearing in the same page.

  • Users are provided a simple interface to enter their search terms and receive results.
  • All results include a description of each returned result.
  • All search results are linked directly to an actual Wikipedia entry.
  • All results are filtered by most relevant to least relevant to the search terms entered.
  • Users are also offered a "Random Search" button on the interface to produce a random search entry.
  • All searches, results and descriptions are responsive to various devices and screen sizes.
Weather App

Weather App

Live Weather Based on Current Location

This web-based app is based on a user's current geolocation coordinates. Users are required to allow permission to check their geolocation via their browser and the app will automatically generate the weather information.

  • Coordinates are generated by the user's browser, when the user allows the webpage permission to get its current location.
  • All weather information is provided by an open weather API.
  • Users don't have to do anything beyond granting location access. The web app is self-working.
  • The temperature is available in both farenheit and celsius measurements. A simple blue button acts as a switch between the two.
  • The background wallpaper will change according to the temperature outside. Bright and warm wallpapers for warmer weather, cooler and darker photos for cooler weather.
  • The app is responsive to all devices and screen sizes.

    Bake Shop UI Design Concept

    Bake Shop UI Design Concept

    This concept design was inspired by a stay-at-home mom who wanted to pursue her own bake shop after completing a pastry program at a NYC culinary school. After researching some other sites for bake shops, cupcakes and pastries, it became apparent that most other sites had a lot of "content clutter" that made the user experience a bit overwhelming. The goal of this design was to simplify the interface so that even an 80 year old grandparent should be able to place an order for cupcakes without much confusion.

  • The tone and concept of the bake shop is set with a large image of cupcakes, along with a friendly font.
  • The menu is immediately available for viewing below the main image. Conceptually, users will want to know about the products first, then the shop's details later.
  • The menu layout is responsive and automatically divides the different types of products available.
  • The background details of the shop is presented in a simple display that doesn't distract from the business' main goal: Showcasing and selling its products.
  • A simple footer without a contact form was selected to encourage potential customers to contact the shop directly with their inquiries. A phone number and location would be provided, so that the potential customers can quickly interact with the shop on a personal level and immediately create a friendly buying experience.
  • Additional content can be added, such as a reviews section or links to a review site like Yelp.
    • Pomodoro Clock

    Pomodoro Clock

    This project is built around the "pomodoro clock" concept that helps increases productivity by intelligently managing time working and taking breaks. The idea that a user can be productive for 25 minutes and break for 5 minutes relieves the user from being fatigued and wanting to stray from their tasks.

  • The clock loads a 25 minute timer by default. A 5 minute break timer is also set.
  • Easy to navigate buttons allows users to increment or decrement the timers by 5 minutes at a time. The shortest of both clocks being 5 minutes, with no maximum time limit.
  • The clock will exhibit a pop-up alert when either timer has completed. A reset button is then presented to the user to restart the timers.
